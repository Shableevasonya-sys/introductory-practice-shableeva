# Краткий конспект по Git

## Основные команды

- `git status` — показать текущее состояние репозитория.
- `git add <file>` — добавить файл в индекс (staging area).
- `git add .` — добавить все изменённые файлы.
- `git commit -m "сообщение"` — зафиксировать изменения в истории.
- `git log --oneline` — краткая история коммитов.
- `git diff` — показать разницу между текущими файлами и индексом/последним коммитом.

## Работа с ветками

- `git branch` — список локальных веток.
- `git branch <name>` — создать новую ветку.
- `git checkout <name>` — переключиться на ветку.
- `git checkout -b <name>` — создать и сразу перейти на новую ветку.
- `git merge <name>` — слить указанную ветку в текущую.
- `git branch -d <name>` — удалить локальную ветку.

## Работа с удалённым репозиторием

- `git remote -v` — показать список удалённых репозиториев.
- `git push` — отправить локальные коммиты на удалённый репозиторий.
- `git pull` — забрать изменения из удалённого репозитория и слить в текущую ветку.
- `git clone <url>` — клонировать удалённый репозиторий.

## Полезные команды

- `git restore <file>` — откатить изменения в файле к последнему коммиту.
- `git reset HEAD <file>` — убрать файл из индекса, не трогая содержимое.
- `git stash` — временно спрятать незакоммиченные изменения.
- `git stash pop` — достать спрятанные изменения.
